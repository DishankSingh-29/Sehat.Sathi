# Documentation

This directory contains project documentation.

## Files

- **api-docs.md**: Complete API documentation with all endpoints, request/response examples, and authentication details.

## Flow Diagram

- **flow-diagram.png**: System architecture and flow diagram showing the interaction between frontend, backend, and database components.

### Usage
The flow diagram illustrates:
- User authentication flow
- Appointment booking process
- Chatbot interaction flow
- Report generation workflow

## Screenshots

The `screenshots/` folder is intended for storing:
- UI screenshots
- API response examples
- System architecture diagrams
- User flow demonstrations

### Adding Screenshots
1. Place screenshots in this folder
2. Name them descriptively (e.g., `login-page.png`, `dashboard-view.png`)
3. Reference them in documentation as needed
